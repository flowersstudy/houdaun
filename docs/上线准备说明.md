# 上线准备说明

## 1. 当前 PDF 存储位置

- 当前代码默认把学生上传的 PDF 文件保存到后端项目目录下的 `uploads/`
- 当前本地路径是：`D:\xitong\houduan\student-teacher\uploads`
- 数据库表 `pdf_submissions` 只保存提交记录和文件名，不保存 PDF 二进制内容

## 2. 推荐的生产环境文件存储方式

- 如果公司有自己的服务器，可以先继续用服务器本地磁盘
- 建议把上传目录配置到项目目录外，例如：
  - Windows：`D:\data\student-teacher\uploads`
  - Linux：`/data/student-teacher/uploads`
- 这样发版时不会误删上传文件

## 3. 生产环境需要配置的环境变量

后端 `.env` 至少需要：

```env
DB_HOST=
DB_PORT=
DB_USER=
DB_PASSWORD=
DB_NAME=

JWT_SECRET=
JWT_EXPIRES_IN=7d

PORT=3000
UPLOADS_DIR=

WX_APPID=
WX_SECRET=
```

说明：

- `UPLOADS_DIR`：PDF 上传目录，不配置时默认使用项目内 `uploads/`
- `WX_APPID / WX_SECRET`：必须和当前小程序对应，否则学生微信登录失败

## 4. 学生端上线前要确认

- 把学生端生产后端域名填到：`C:\Users\15012\草稿\student-app\utils\runtime-config.js`
- 小程序后台配置：
  - `request合法域名`
  - `uploadFile合法域名`
  - `socket合法域名`
- 域名必须是 HTTPS，Socket 需要支持 WSS

## 5. 老师端上线前要确认

- 老师端生产 API 地址由 `VITE_API_BASE_URL` 控制
- 若前后端同域部署，可通过反向代理把 `/api` 转发到后端
- 若前后端分域部署，需要把生产 API 地址写入 `.env.production`

## 6. 后端上线前要确认

- MySQL 已初始化并执行过建表/种子脚本
- 后端进程可正常访问数据库
- 上传目录存在且具备读写权限
- 上传目录已纳入备份计划
- `3000` 端口或反向代理已开放

## 7. 建议的上线检查顺序

1. 后端 `.env` 配好并启动
2. 访问 `/health` 确认服务正常
3. 老师账号登录成功
4. 学生微信登录成功
5. 学生上传 PDF 成功
6. 老师端能看到待批改列表
7. 老师端能打开 PDF 并保存批改结果
8. 请假、聊天、首页进度确认可用

## 8. 备份建议

- 数据库定期备份
- `UPLOADS_DIR` 定期备份
- 备份时数据库和上传目录都要保留，否则会出现“有记录但文件丢失”
